# XiaoBa World 世界观定义

## 1. XiaoBa World 是什么

`XiaoBa World` 不是“很多 prompt 的集合”，也不是“几个角色皮肤”。

它的本质是：

- 一个统一的 runtime 内核
- 一组具有明确职责边界的 agent 角色
- 一套让这些角色持续协作、自我优化、持续进化的运行秩序

所以 `XiaoBa World` 的重点不是“角色多”，而是：

- 每个角色为什么存在
- 每个角色负责什么，不负责什么
- 角色之间如何交接
- 系统如何从真实用户行为中不断进化

一句话定义：

`XiaoBa World = 一个以统一 runtime 为核心、以角色分工为组织形式、以日志回流驱动自我进化的多 agent 世界。`

## 2. 世界观核心命题

### 2.1 XiaoBa 不是单个助手，而是一个世界

一个单体助手只能“尽量什么都做一点”。

而 `XiaoBa World` 认为：

- 真实复杂任务不该由一个 agent 长期独吞
- 不同任务需要不同视角、不同纪律、不同表达方式
- “协作”本身是一等能力，不只是附属功能

所以在这个世界里：

- `角色` 是天然存在的
- `协作` 是系统默认能力
- `进化` 是系统核心目标

### 2.2 真实日志高于抽象设想

`XiaoBa World` 的进化，不依赖空想，不依赖闭门造车。

它的第一原则是：

`真实运行日志 > 主观想象 > 漂亮概念`

也就是说：

- 新 skill 从真实重复模式里长出来
- 新 role 从真实稳定职责里长出来
- runtime 优化从真实失败里长出来

这就是为什么 `InspectorCat` 是这个世界里的核心角色之一。

### 2.3 统一内核，分化角色

`XiaoBa World` 坚持一条工程底线：

- 只维护一套 `XiaoBa-CLI` runtime 内核
- 不为每个角色复制一套 runtime

角色的分化体现在：

- `system prompt`
- `skills`
- `tools`
- `utils`
- `runtime support`

而不是体现在“各自 fork 一份 XiaoBa”。

## 3. 世界中的三类存在

`XiaoBa World` 里有三种基本存在。

### 3.1 Runtime

Runtime 是世界的物理层。

它负责：

- 消息收发
- 工具执行
- skill 调度
- 日志记录
- 平台接入
- 角色加载

它不负责“人格”，也不负责“领域判断”。

### 3.2 Role

Role 是世界里的“职业”。

每个 role 都应该有：

- 明确存在理由
- 明确职责边界
- 明确输入输出
- 明确与其他角色的关系

role 不是换个名字，而是换一种稳定工作方式。

### 3.3 Loop

Loop 是世界的生命机制。

它定义：

- 角色如何从日志中学习
- 系统如何发现问题
- 问题如何被修复、验证、发布

没有 loop，`XiaoBa World` 只是角色目录。
有了 loop，它才是一个会成长的世界。

## 4. 世界的两套协作秩序

这是 `XiaoBa World` 最重要的结构。

这个世界同时拥有两套协作秩序：

1. `工作流秩序`
2. `群聊秩序`

它们不是冲突关系，而是分层关系。

### 4.1 工作流秩序

工作流秩序负责：

- 推进状态
- 交付结果
- 串联角色
- 控制重试、验证、发布

它适合：

- `InspectorCat -> RuntimeDoctor -> SkillDoctor -> Verifier -> Publisher`
- 任何需要明确阶段和明确验收的事情

一句话：

`工作流是 XiaoBa World 的骨架。`

### 4.2 群聊秩序

群聊秩序负责：

- 角色之间可见协作
- 角色讨论和辩论
- 人类旁观和介入
- 非强约束型判断

它适合：

- 讨论边界
- 辩论方案
- 旁听群聊
- 角色之间做轻量交接

一句话：

`群聊是 XiaoBa World 的界面与社交层。`

### 4.3 两者的边界

必须明确：

- 该推进状态的事，走工作流
- 该表达观点的事，走群聊

不能反过来：

- 不要靠群聊消息决定核心状态推进
- 不要把所有讨论都硬塞成流程节点

## 5. 这个世界为什么合理

`XiaoBa World` 之所以成立，是因为真实 AI 系统天然会分裂出三种需求：

1. `执行需求`
2. `审查需求`
3. `进化需求`

单角色系统擅长做执行，但很难长期兼顾审查与进化。

所以这个世界不是人为搞复杂，而是在顺着真实复杂度生长。

## 6. 角色观

### 6.1 角色不是人格皮肤

在 `XiaoBa World` 里，角色首先是“职责容器”，其次才是表达风格。

也就是说：

- `InspectorCat` 的本质不是“像猫一样说话”
- 而是“它专门负责 review 和问题归因”

### 6.2 角色必须有边界

一个角色只有在下面这些问题都能回答清楚时，才算成立：

1. 它为什么存在
2. 它优先解决什么
3. 它不该解决什么
4. 它和其他角色的分界线是什么
5. 它的交付物是什么

如果这些问题答不清楚，那它就不是角色，只是壳。

### 6.3 角色从真实工作流中生长

`XiaoBa World` 里的角色不是模板批量生成的。

角色应该来自：

- 长日志
- 稳定重复职责
- 清晰的判断框架
- 明确的交付物

所以：

- `InspectorCat` 来自 review 职责
- `SciPaperDoctor` 来自论文交付职责

未来其他角色也应该这么长出来。

## 7. 角色谱系

这个世界里的角色，建议分成三个谱系。

### 7.1 守护者谱系

这些角色负责系统稳定、自我进化、自我修复。

例如：

- `InspectorCat`
- `RuntimeDoctor`
- `SkillDoctor`
- `VerifierCat`
- `Publisher`

它们对应世界的“免疫系统”和“新陈代谢系统”。

### 7.2 工匠谱系

这些角色负责把模式固化成可复用能力。

例如：

- `SkillDoctor`
- `RoleArchitect`

它们对应世界的“制造系统”。

### 7.3 领域医生谱系

这些角色负责具体业务领域的执行与交付。

例如：

- `SciPaperDoctor`

未来可扩展：

- `CodeDoctor`
- `DataDoctor`
- `OpsDoctor`

它们对应世界的“生产系统”。

## 8. XiaoBa World 的基本法

建议把下面这些当成世界基本法。

### 法一：统一 runtime 法

`XiaoBa World` 只允许一个 runtime 内核。

### 法二：真实日志法

真实日志是新 skill、新 role、runtime 修复的最高依据。

### 法三：边界优先法

任何角色在扩张之前，都必须先定义边界。

### 法四：交付物法

角色必须产出明确交付物，不能只给情绪化意见。

### 法五：讨论与执行分层法

群聊负责讨论，工作流负责执行。

### 法六：默认先审查法

系统角色默认先 review，再修改，再发布。

### 法七：可回流法

所有重要行动都必须能在日志和报告中回流，成为下一轮进化素材。

## 9. XiaoBa World 的时间观

这个世界不是一次性构建完成的，它是分阶段长出来的。

### 第一阶段：单体 XiaoBa

特点：

- 一个 runtime
- 一个主助手
- 多数事情混在一起做

### 第二阶段：角色分化

特点：

- 从真实日志里分化出第一批角色
- 例如 `InspectorCat`、`SciPaperDoctor`

### 第三阶段：loop 成型

特点：

- 角色开始形成闭环
- 系统能自我审查、自我修复、自我提炼

### 第四阶段：world 成熟

特点：

- 既有系统角色，也有领域角色
- 既有工作流，也有群聊协作
- 人类不是系统外部，而是在系统中持续参与

## 10. 人类在这个世界中的位置

人类不是被 XiaoBa World 替代的人。

人类在这个世界里有三重身份：

1. `使用者`
2. `观察者`
3. `裁决者`

### 10.1 使用者

人类通过各个平台使用 XiaoBa 角色完成任务。

### 10.2 观察者

人类可以在群聊中看到角色如何协作、如何交接、如何争论。

### 10.3 裁决者

最终：

- 是否接受一个修复
- 是否发布一个 skill
- 是否正式立一个新角色

这些关键决定仍然由人类把关。

所以 `XiaoBa World` 不是无人世界，而是人机共治世界。

## 11. 群聊在这个世界里的意义

群聊不是装饰，不是 UI 皮肤。

群聊的世界意义是：

- 让协作可见
- 让角色有共同场域
- 让人类能随时旁听和插话

在这个世界里，群聊像“议事厅”。

而工作流像“行政系统”。

两者缺一不可。

## 12. 世界的终极目标

`XiaoBa World` 的终极目标不是“角色越来越多”，而是：

- 角色越来越有边界
- runtime 越来越稳
- skill 越来越准
- 新角色从真实工作中自然长出
- 人类越来越像在使用一个会成长的组织，而不是一个死板工具

一句话收束：

`XiaoBa World` 想成为的，不是一个大号 AI 助手，而是一个由统一内核驱动、由角色分工组织、由真实日志反哺成长的人机协作世界。`

## 13. 当前世界观下的落地结论

按这个世界观，当前最合理的推进顺序是：

1. 先把 `InspectorCat` 打磨成熟
2. 造出 `RuntimeDoctor`
3. 造出 `SkillDoctor`
4. 造出 `VerifierCat`
5. 让 loop 先跑起来
6. 再继续扩领域 doctor

也就是说：

- 先让这个世界活起来
- 再让这个世界变丰富

而不是反过来。
